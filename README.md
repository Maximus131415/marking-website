# marking-website
# 📌 Функционал модулей

## **1.1. Веб-интерфейс (Frontend)**

Пользовательский интерфейс сайта для работы с разметкой данных.

**Основной функционал:**
* Страница входа и регистрации
  * Выбор роли: заказчик / исполнитель
* Личный кабинет
  * Заказчик: список созданных заказов, их статус
  * Исполнитель: список взятых задач, прогресс
* Окно оформления заказа
  * Загрузка изображений
  * Выбор типа разметки:
    * Классификация
    * Детекция (bounding boxes)
  * Указание классов
* Каталог заказов
  * Список доступных задач для исполнителей
  * Возможность взять заказ в работу
* Интерфейс разметки
  * Отображение изображения
  * Выбор класса (classification)
  * Рисование bounding boxes (детекция)
  * Сохранение результата

---

## **1.2. Backend & API (Серверная логика)**

Связующее звено между фронтендом, базой данных и логикой системы.
**Функционал:**
* Обработка запросов от frontend
* Авторизация и аутентификация пользователей 
* Работа с заказами:
  * Создание заказов
  * Назначение исполнителей
  * Отслеживание статуса
* Работа с разметкой:
  * Сохранение результатов разметки
  * Передача изображений
* API для всех операций (users, orders, annotations)

---

## **1.3. Модуль разметки (Annotation Logic)**

Логика работы с разметкой изображений.
**Функционал:**
* Классификация:
  * Присвоение одного или нескольких классов изображению
* Детекция:
  * Создание bounding boxes
  * Привязка классов к каждому боксу
* Проверка корректности разметки
* Формирование JSON-разметки 

---

## **1.4. Данные и Админка**
Хранение и управление данными проекта.
**Функционал:**
* Хранение:
  * Пользователей
  * Заказов
  * Изображений
  * Разметки
* Админ-панель / дашборд:
  * Количество заказов
  * Количество размеченных изображений
  * Активность исполнителей
* Экспорт размеченных данных
